# Spec Workflow Checklist

## Read first

- `docs/technical-assignment.md`
- `docs/project-spec.md`

## Lock before planning

- user goal
- affected subsystem
- public contract touched
- acceptance criteria
- assumptions

## Reject immediately if proposed

- target other than weekly revenue by `City`
- single split validation
- future-aware features
- rolling features without `shift(1)`
- autoregressive inference

## Preferred output shape

1. Objective
2. Invariants
3. Planned change
4. Acceptance checks
5. Risks and handoffs
